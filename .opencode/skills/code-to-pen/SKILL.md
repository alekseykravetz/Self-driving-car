# Code → .pen Reverse Engineering Workflow

Recreates the visual design of a web application as a Pencil `.pen` design file by analyzing the source code (HTML, CSS, TypeScript) and capturing runtime screenshots with Playwright.

## When to use

- The user asks you to generate or regenerate an `app.pen` (or any `.pen`) file from an existing web project.
- You need to add new screen frames that match the actual rendered HTML pages.
- The existing `.pen` file is stale, generic, or was generated by a model weak at visual design.

## Workflow

### Step 1 — Discover entry points

Use `glob` to find all HTML files in the project:

```
html/**/*.html
index.html
```

Read each one. Note:

- The `<title>` tag (becomes the screen name)
- The custom elements / web components used (`<training-panel>`, `<traffic-panel>`, etc.)
- The canvas elements (`<canvas id="gameCanvas">`, `<canvas id="networkCanvas">`, etc.)
- The main `<script>` entry point (`src="/js/simulator/entry.js"`)
- The CSS files (`<link rel="stylesheet" href="...">`)

### Step 2 — Extract CSS design tokens

Read the project's CSS entry points: `styles/tokens.css` (design tokens — the single source of truth), `styles/index.css` (shared core), and the page entries `styles/simulator.css`, `styles/landing.css`, `styles/race.css`, `styles/world.css`. The `styles/` folder follows Atomic Design: `atoms/` → `molecules/` → `organisms/` → `templates/` → `pages/`. Extract all design tokens as Pencil variables:

| CSS concept                        | Pencil equivalent                   |
| ---------------------------------- | ----------------------------------- |
| `background: #0f0f14`              | `fill: "#0f0f14"` or `$bg` variable |
| `color: #e8e8e8`                   | `fill: "#e8e8e8"` for text          |
| `font-family: 'Inter', sans-serif` | `fontFamily: "Inter"`               |
| `font-size: 13px`                  | `fontSize: 13`                      |
| `font-weight: 600`                 | `fontWeight: "600"`                 |
| `border-radius: 8px`               | `cornerRadius: 8`                   |
| `gap: 8px`                         | `gap: 8`                            |
| `padding: 12px`                    | `padding: 12`                       |
| `opacity: 0.6`                     | `opacity: 0.6`                      |

For design tokens, set Pencil variables via `SetVariables()`:

```js
SetVariables({
  'bg-app': { type: 'color', value: '#0f0f14' },
  'bg-surface': { type: 'color', value: 'rgba(15,15,20,0.92)' },
  'bg-overlay': { type: 'color', value: '#00000099' },
  'bg-canvas': { type: 'color', value: '#2a5' },
  'bg-card': { type: 'color', value: 'rgba(255,255,255,0.04)' },
  'text-primary': { type: 'color', value: '#e8e8e8' },
  'text-secondary': { type: 'color', value: '#888888' },
  'text-muted': { type: 'color', value: 'rgba(255,255,255,0.5)' },
  'border-subtle': { type: 'color', value: 'rgba(255,255,255,0.08)' },
  'border-visible': { type: 'color', value: '#ffffff26' },
  'accent-green': { type: 'color', value: '#7ddf7d' },
  'accent-blue': { type: 'color', value: '#5cb8ff' },
  'accent-yellow': { type: 'color', value: '#ffcc44' },
  'font-ui': { type: 'string', value: 'Arial' },
  'font-mono': { type: 'string', value: "'Courier New', monospace" },
  'space-1': { type: 'number', value: 4 },
  'space-2': { type: 'number', value: 8 },
  'space-3': { type: 'number', value: 12 },
  'space-4': { type: 'number', value: 16 },
  'radius-sm': { type: 'number', value: 3 },
  'radius-md': { type: 'number', value: 5 },
  'radius-lg': { type: 'number', value: 8 },
  'radius-xl': { type: 'number', value: 12 },
});
```

Token names in `styles/tokens.css` already follow the `--color-*`, `--space-*`, `--text-*`, `--radius-*` convention; map each to a Pencil variable by dropping the leading `--` (the category prefix is optional).

### Step 3 — Capture visual reference with Playwright

Use Playwright to take screenshots of every page. Even for canvas-heavy apps, screenshots provide essential layout context.

```js
// Start a browser, navigate, screenshot each page
await page.goto('http://localhost:9090/');
await page.screenshot({
  path: 'screenshot-landing.png',
  fullPage: true,
  type: 'png',
});

await page.goto('http://localhost:9090/html/simulator.html');
await page.screenshot({ path: 'screenshot-simulator.png', type: 'png' });

// etc for all entry points
```

Also capture the accessibility snapshot for DOM structure:

```js
// Get the page accessibility/DOM tree
// Use playwright_browser_snapshot() tool
```

Store screenshots as reference for color, spacing, and layout decisions.

### Step 4 — Plan the .pen structure

Each HTML entry point becomes a **top-level frame** inside a container frame named "📱 App Screens". Structure:

```
document
├── 📱 App Screens (frame, clip: true, layout: "none")
│   ├── 🏠 Landing Page (frame, clip: true)
│   ├── 🧠 Simulator (World Mode) (frame, layout: "none")
│   ├── 🧠 Simulator (Simple Mode) (frame)
│   ├── 🧠 Simulator — Training Init Modal (frame)
│   ├── 🎓 Human Training (World) (frame)
│   ├── 🎓 Human Training (Simple) (frame)
│   ├── 🎓 Human Training — Config Modal (frame)
│   ├── 🏁 Race (Keyboard) (frame)
│   ├── 🏁 Race (Camera) (frame)
│   ├── 🏁 Race (Phone) (frame)
│   ├── 🗺️ World Editor (frame)
│   ├── 🚦 Live Traffic Jam (frame)
│   └── 📋 Page Index (frame)
├── ⚙️ Components Library (frame, layout: "none")  // reusable components only
│   ├── ToolbarBtn (reusable: true)
│   ├── StatRow (reusable: true)
│   ├── ... (only components actually needed by screens)
```

**Rules:**

- Each screen is a top-level frame (1400×900 for standard, proportional for landing).
- Use `clip: true` on screen frames to prevent overflow.
- Keep only 1-2 component frames with reusable components needed by the screens.
- Do NOT place text, icons, buttons, cards directly in `document`.

### Step 5 — Build each screen

For each screen, analyze the HTML to determine:

1. **Layout containers** — `div#simulatorLayout`, `#rightPanel`, `#simulatorToolbar`
2. **Canvas elements** — `canvas#gameCanvas`, `canvas#networkCanvas`, `canvas#miniMapCanvas`
3. **Custom elements** — `<training-panel>`, `<world-toolbar>`, etc.
4. **Toolbar groupings** — world-toolbar, layout-toolbar, animation-loop-toolbar, shortcuts-toolbar

Map to Pencil frames:

```html
<div id="simulatorLayout">
  <canvas id="gameCanvas"></canvas>
  <canvas id="cameraCanvas"></canvas>
  <div id="rightPanel">
    <canvas id="networkCanvas"></canvas>
    <canvas id="miniMapCanvas"></canvas>
  </div>
  <training-panel></training-panel>
</div>
```

Becomes:

```js
// Screen: Simulator
screen = Insert(document, {
  type: "frame", name: "🧠 Simulator", x: ..., y: ...,
  width: 1400, height: 900,
  clip: true, fill: "$bg-app", layout: "none"
});

// Game canvas (takes most of the space)
Insert(screen, {
  type: "frame", name: "GameCanvas",
  width: 800, height: 900,
  fill: "#2a5", x: 0, y: 0
});

// Camera canvas (strip on the right of game)
Insert(screen, {
  type: "frame", name: "CameraCanvas",
  width: 200, height: 900,
  fill: { type: "gradient", ... }, x: 800, y: 0
});

// Right panel (network + minimap)
rightPanel = Insert(screen, {
  type: "frame", name: "RightPanel",
  width: 200, height: 900,
  layout: "vertical", fill: "#000000", x: 1000, y: 0
});
Insert(rightPanel, {
  type: "frame", name: "NetworkCanvas",
  width: "fill_container", height: 600,
  fill: "#000000"
});
Insert(rightPanel, {
  type: "frame", name: "MiniMapCanvas",
  width: "fill_container", height: 300,
  fill: "#000000"
});

// Training panel (rightmost)
Insert(screen, {
  type: "frame", name: "TrainingPanel",
  width: 200, height: 900,
  fill: "$bg-panel", layout: "vertical",
  padding: 8, x: 1200, y: 0
});

// Toolbar (floating at top-left)
toolbar = Insert(screen, {
  type: "frame", name: "SimulatorToolbar",
  layout: "vertical", gap: 6, x: 10, y: 10
});
```

### Step 6 — Create reusable components

Extract repeated UI patterns into reusable components:

```js
// Toolbar button (32×32, dark bg, rounded)
ToolbarBtn = Insert(componentsFrame, {
  type: 'frame',
  name: 'ToolbarBtn',
  reusable: true,
  width: 32,
  height: 32,
  fill: '#ffffff12',
  cornerRadius: 5,
  justifyContent: 'center',
  alignItems: 'center',
  children: [
    {
      type: 'text',
      name: 'Icon',
      fill: '#ffffff',
      content: '🔘',
      fontFamily: 'Inter',
      fontSize: 16,
    },
  ],
});
```

Common components to extract:

- **ToolbarBtn** — 32×32 icon button for toolbar
- **BtnLg** — full-width action button (38px height, 8px radius)
- **StatRow** — emoji + label + value in a row
- **SectionTitle** — uppercase label with letter-spacing
- **KeyIndicator** — key binding display (mono font)
- **TrafficCarRow** — traffic simulation car listing
- **ModalOverlay** — modal background with content area

### Step 7 — Verify

After building screens, verify each one:

1. Open the `.pen` file in Pencil desktop or via `batch_get`
2. Check that all screen frames exist and aren't collapsed
3. Verify layout proportions match the real HTML (canvas sizes, panel widths)
4. Check color contrast and font choices match the CSS
5. Ensure component instances are properly wired

### CSS → Pencil property mapping reference

| HTML/CSS                                 | Pencil                                |
| ---------------------------------------- | ------------------------------------- |
| `display: flex; flex-direction: column;` | `layout: "vertical"`                  |
| `display: flex; flex-direction: row;`    | `layout: "horizontal"` (default)      |
| `gap: 8px`                               | `gap: 8`                              |
| `padding: 8px`                           | `padding: 8`                          |
| `padding: 4px 12px`                      | `padding: [4, 12]`                    |
| `padding: 4px 8px 4px 12px`              | `padding: [4, 12, 4, 8]`              |
| `justify-content: center`                | `justifyContent: "center"`            |
| `justify-content: space-between`         | `justifyContent: "space_between"`     |
| `align-items: center`                    | `alignItems: "center"`                |
| `flex: 1`                                | `width: "fill_container"`             |
| `border-radius: 8px`                     | `cornerRadius: 8`                     |
| `border: 1px solid rgba(...)`            | `stroke: "rgba(...)", strokeWidth: 1` |
| `overflow: hidden`                       | `clip: true`                          |
| `font-family: 'Inter'`                   | `fontFamily: "Inter"`                 |
| `font-size: 13px`                        | `fontSize: 13`                        |
| `font-weight: 700`                       | `fontWeight: "700"`                   |
| `letter-spacing: 1px`                    | `letterSpacing: 1`                    |
| `color: #e8e8e8`                         | `fill: "#e8e8e8"`                     |
| `text-transform: uppercase`              | just use uppercase in `content`       |
| `background: rgba(0,0,0,0.6)`            | `fill: "#00000099"` (hex with alpha)  |
| `box-shadow`                             | `effect: { type: "shadow", ... }`     |

### Important constraints

- **No percentage values** — never use `"100%"`, `"50%"` for width/height. Use `"fill_container"` or explicit pixel values.
- **No `margin`** — use `gap` on the parent + `padding` on containers.
- **`x`/`y` only when `layout: "none"`** — children in flex layout ignore x/y.
- **Text needs `textGrowth` for wrapping** — use `"fixed-width"` + explicit `width` for multi-line text.
- **Text is invisible without `fill`** — always set `fill` on text (even emoji).
- **Components use `reusable: true`** — instances use `type: "ref"` with `ref` pointing to the component ID.
- **`slot: false` for disabled slots** — don't use `enabled: false` on frames with children.
- **`FindEmptySpace` before every new top-level node** — never overlap root objects.
